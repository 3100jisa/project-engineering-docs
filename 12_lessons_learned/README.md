# Lessons Learned

> This section captures lessons learned across all project disciplines to enable continuous improvement.

## Purpose

Lessons learned are structured observations from completed work that help:
- Avoid repeating past mistakes
- Scale successful approaches
- Feed process improvement activities (ASPICE PIM.3)

## How to Contribute

1. Copy the [lessons_learned_template.md](lessons_learned_template.md)
2. Fill in all fields — be specific and factual
3. Place your entry in the appropriate discipline file (e.g., `ll_software_engineering.md`)
4. Submit via pull request

## Discipline Files

| File | Scope |
|---|---|
| `ll_system_engineering.md` | SYS processes, requirements, architecture |
| `ll_software_engineering.md` | SWE processes, code, SW testing |
| `ll_firmware_engineering.md` | Embedded FW, BSP, RTOS, drivers |
| `ll_hardware_engineering.md` | HWE processes, schematics, PCB, testing |
| `ll_project_management.md` | Planning, risk, estimation, communication |
| `ll_safety_and_compliance.md` | Functional safety, audits, compliance |

## Review Cadence

Lessons learned should be reviewed at:
- End of each project phase
- Post-release retrospective
- Annual process audit
